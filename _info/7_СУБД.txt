Реляционные системы управления базой данных:
1. Oracle 
+ Надежность, скорость, умение работать с базами данных очень большого размера
- высокая цена
1) Не существует практики использования Oracle для маленьких баз данных. Крупные организации могут    позволить себе эту базу данных. 
2) Имеет сложные средства администрирования. Надо изучать специально. 

2. PostgreSQL
+ Надежен, работает с большими данными
  Есть практика работы с любым размером базы данных (так как свободный продукт)
- удобные средства администрирования нужно приобретать отдельно, но они стоят не дорого.
Это единственный недостаток
postgresql.org - скачать для 32 разрядов, так как Qt у нас 32-разрядный

3. Microsoft SQL Server
Он работает только на Windows
Имеет бесплатную версию MS SQL Express
- размер одной базы данных ограничен 4 гб
Для большинства случаев это ограничение не принципиально
Фича в следующем: не пытаться пользоваться экспрессом без полной версией. 
Главный офис, где работает полная версия, и филиалы, где работает экспресс.
Знать полную версию MS SQL нужно обязательно

4. MySQL
Применяется для интернета главным образом
Быстро работает на чтение, но медленно на запись и идентификацию данных. 
База спецназначения. 

-------------------------------------------

5. SQLite 
Запрещается использовать в качестве основной. 
Она встраиваемая. Может использоваться для выполнения некоторых задач на стороне клиента.
Может сослужить хорошую службу для вспомогательных данных.
Например скопировать данные из другой базы в неё.
Также формат обмена данными.
Перекачать по сети и скопировать в другую базу данных.
Хранить основные данные в SQLite нельзя
